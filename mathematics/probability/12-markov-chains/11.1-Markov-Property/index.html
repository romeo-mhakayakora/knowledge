    What Is a Markov Chain?¶

    A Markov chain is a sequence of random variables $$ X_0, X_1, X_2, X_3, \ldots $$ whose future evolution depends only on the present state.

    The key point is not that there is no dependence. There usually is dependence. The key point is that the dependence is organized so that once the present state is known, older history gives no extra information for predicting the next step.

    Think of the chain as having memory only of where it is right now. It does not remember how it got there.

    A useful visual is:

    General process:
X_0 -> X_1 -> X_2 -> ... -> X_n -> X_{n+1}
|      |      |            |      |
v      v      v            v      v
(history can matter)

Markov chain:
X_0 -> X_1 -> ... -> X_{n-1} ~~X_n~~~~> X_{n+1}
                                |
                                v
                     next step uses only X_n

    Scouting Report¶

    What to look for: a process indexed by time where the next step is determined by a compact current state.

    Why you care: this is the structural condition that makes transition matrices and matrix powers possible.

    Why This Is Defined This Way¶

    The definition is designed so that the “state” contains exactly the information needed to predict the next move. If the present state were not enough, then the next-step rule would have to depend on some longer history, and the process would no longer be captured by one-step transition probabilities alone.

    This is the compression principle behind Markov chains: a full path is reduced to a single current state.

    What Would Break Without This Property?¶

    If the next-step probability depends on older states too, then the process is not Markov in the current state space. For example, if tomorrow’s weather depends on today and yesterday, then today alone is not enough information. The state would have to be enlarged, perhaps to pairs like \((X_{n-1}, X_n)\).

    Without the Markov property, there is no clean one-step matrix that captures the dynamics.

    Phenomenon Metadata¶

    Element Purpose
    Structural Signature The next move depends only on the current state
    Core Invariant The current state is sufficient for one-step prediction
    Compression Handle “The past is folded into the present”
    Boundary / Failure Mode If older history changes the next-step law, the state is too small
    Phenomenon Web See the Transition Matrix Q, The n-Step Transition Matrix Qn, and the Chapman-Kolmogorov Equation blocks

    Why Markov Chains Matter¶

    Many systems evolve over time: weather, routing, queues, populations, and reinforcement learning environments.

    The challenge is that history grows forever. If we had to condition on the entire past, then the number of possible histories would expand rapidly with time. That makes prediction and computation unwieldy.

    The Markov property solves this by saying that the current state already contains everything relevant about the past.

    Intuition: instead of storing a whole trajectory, we store one state.

    Why This Matters for Computation¶

    If the state space has \(M\) states, then a history of length \(n+1\) has \(M^{n+1}\) possible sequences. That is enormous for large \(n\).

    With the Markov property, we only need to track the \(M\) current states. That is the difference between a growing history table and a fixed-size transition matrix.

    What Would Break Without It?¶

    Without the Markov property, the conditional probability of the next step could depend on the full history, so there would be no single transition matrix indexed only by current state and next state.

    Phenomenon Metadata¶

    Element Purpose
    Structural Signature History grows but prediction still depends only on the present
    Core Invariant The current state is a sufficient statistic for the future
    Compression Handle “One state replaces a whole trajectory”
    Boundary / Failure Mode If memory of the past improves prediction, Markov fails
    Phenomenon Web Explains why matrices replace history tables

    State Space¶

    The state space is the set of all possible states of the chain. We usually denote it by \(S\).

    Examples: - \(S=\{1,2,3\}\) - \(S=\{\text{Sunny}, \text{Cloudy}, \text{Rainy}\}\) - \(S=\{0,1,2,\ldots,10\}\)

    Every random variable \(X_n\) takes values in \(S\).

    Scouting Report¶

    What to look for: the complete list of values that the chain can occupy.

    Why you care: the state space determines the row and column labels of the transition matrix.

    Why This Definition Exists¶

    The state space tells us what values the indices \(i\) and \(j\) may take in $$ q_{ij}=P(X_{n+1}=j \mid X_n=i). $$ Without a specified state space, the matrix dimensions are undefined.

    What Would Break Without It?¶

    If \(S\) were not specified, we would not know how large the transition matrix is, which entries are valid, or how to compute matrix powers consistently.

    For infinite state spaces, the theory changes substantially: the matrix may become infinite-dimensional, and uncountable spaces require kernels and measure theory rather than finite matrices.

    Phenomenon Metadata¶

    Element Purpose
    Structural Signature A well-defined set of possible states
    Core Invariant Every \(X_n\) must take values in the state space
    Compression Handle “The alphabet of the chain”
    Boundary / Failure Mode If the state space is unspecified, the matrix is undefined
    Phenomenon Web Supplies the row and column labels for Q

    Time-Homogeneous Markov Chains¶

    A Markov chain is time-homogeneous if the one-step transition probabilities do not depend on time: $$ q_{ij}=P(X_{n+1}=j \mid X_n=i) $$ for every \(n\).

    So the same transition rule applies at every step.

    Scouting Report¶

    What to look for: the same one-step law at every time step.

    Why you care: one fixed matrix \(Q\) can describe the entire chain only when this property holds.

    Why This Definition Exists¶

    Time-homogeneity lets us use one matrix for all time steps. If the transition rule changed with time, we would need a different matrix at each step: $$ Q_0, Q_1, Q_2, \ldots $$ and then long-run evolution would be a product of different matrices rather than a power of one matrix.

    What Would Break Without It?¶

    If the chain is time-inhomogeneous, then \(Q^n\) no longer describes \(n\)-step motion. Instead we would need a product like $$ Q_0Q_1Q_2\cdots Q_{n-1}. $$ That destroys the neat matrix-power structure.

    Phenomenon Metadata¶

    Element Purpose
    Structural Signature The transition law is constant across time
    Core Invariant The same one-step matrix applies at each step
    Compression Handle “Same physics at every time”
    Boundary / Failure Mode If the rule changes with time, powers of one matrix fail
    Phenomenon Web Enables Qn, Chapman-Kolmogorov, and stationary distributions

    The Markov Property¶

    For all states \(i_0,i_1,\ldots,i_{n-1},i,j\), the Markov property states $$ P(X_{n+1}=j \mid X_n=i, X_{n-1}=i_{n-1}, \ldots, X_0=i_0) = P(X_{n+1}=j \mid X_n=i). $$

    Scouting Report¶

    What to look for: a conditional probability where the whole history appears on one side and only the present appears on the other.

    Why you care: this is the identity that allows multi-step probabilities to be reduced to matrix multiplication.

    Intuition Before the Proof¶

    Once the current state is known, the old path that led there becomes irrelevant for the next move. The current state is the whole summary of the past that matters.

    Why This Definition Exists¶

    If the future depended on the entire past, then the process could not be represented by a transition matrix indexed only by current and next states. The Markov property is exactly what allows the future to be summarized by the present.

    What Would Break Without It?¶

    If the next-step probability still depended on earlier states, then the factorization used in multi-step proofs would not collapse to a single matrix product. The chain would need a larger state space or a more complicated model.

    Phenomenon Metadata¶

    Element Purpose
    Structural Signature Conditioning on the full past equals conditioning on the present alone
    Core Invariant Older history adds no predictive power once the present is fixed
    Compression Handle “History collapses to the current state”
    Boundary / Failure Mode If the past changes the next-step law, the process is not Markov
    Phenomenon Web The engine behind Qn and Chapman-Kolmogorov

    The Transition Matrix Q¶

    The transition matrix is $$ Q=(q_{ij}), $$ where $$ q_{ij}=P(X_{n+1}=j \mid X_n=i). $$

    Rows correspond to the current state. Columns correspond to the next state.

    Why Rows Sum to 1¶

    Fix a current state \(i\). Then the possible next states are \(1,2,\ldots,M\), and exactly one of them must occur. Therefore the events $$ {X_{n+1}=1},{X_{n+1}=2},\ldots,{X_{n+1}=M} $$ form a partition of the sample space conditioned on \(X_n=i\). Hence $$ \sum_{j=1}^M q_{ij}=1. $$

    Scouting Report¶

    What to look for: a nonnegative matrix whose rows sum to 1.

    Why you care: this is the compact object that stores the one-step dynamics.

    Why This Definition Exists¶

    The matrix is arranged so that the \(i\)th row is the full probability distribution of the next state given current state \(i\). That makes the matrix directly interpretable in probabilistic terms.

    What Would Break Without It?¶

    If rows did not sum to 1, then the matrix would not represent a valid probability law. Probability mass would either disappear or exceed 1, and the chain would not be stochastic.

    Phenomenon Metadata¶

    Element Purpose
    Structural Signature Nonnegative matrix with row sums equal to 1
    Core Invariant Each row is a probability distribution
    Compression Handle “Row = current, column = next”
    Boundary / Failure Mode If rows do not sum to 1, the matrix is not stochastic
    Phenomenon Web Supplies the one-step probabilities used in Qn

    How to Read Q in Plain English¶

    To read \(Q\), fix the current state and look across that row. The row tells you the probabilities of the possible next states.

    For instance, if the chain is in state \(3\), then the third row tells you what happens next.

    Notation warning: some books write the transition matrix as \(P\) instead of \(Q\). Here \(Q\) is used to avoid confusion with probability itself. Both notations mean the same thing in this context.

    Row-vector convention: we write distributions as row vectors and multiply on the right: $$ p^{(n+1)}=p^{(n)}Q. $$ If we instead use column vectors, then $$ p_{\text{col}}^{(n+1)}=Q^T p_{\text{col}}^{(n)}. $$ Both conventions are equivalent if used consistently.

    The n-Step Transition Matrix Qn¶

    The matrix \(Q^n\) contains the probabilities of moving in exactly \(n\) steps. Its \((i,j)\) entry is written $$ q_{ij}^{(n)}=P(X_n=j \mid X_0=i). $$

    The parentheses in \(q_{ij}^{(n)}\) matter. They indicate the \((i,j)\) entry of \(Q^n\), not the scalar \((q_{ij})^n\).

    Scouting Report¶

    What to look for: a matrix power on the left and an \(n\)-step conditional probability on the right.

    Why you care: this is the bridge between linear algebra and probability.

    Intuition Before the Theorem¶

    It is tempting to guess that one step is \(Q\), two steps is \(Q^2\), three steps is \(Q^3\), and so on. The theorem below shows why that guess is correct.

    Theorem: Entries of Qn Are n-Step Transition Probabilities¶

    Statement. For every \(n\ge 1\) and all states \(i,j\), $$ (Q^n)_{ij}=P(X_n=j \mid X_0=i). $$

    Scouting Report¶

    What to look for: an \(n\)th matrix power on one side and a conditional probability of an \(n\)-step transition on the other.

    Why you care: this theorem turns matrix powers into a probability calculator.

    Full Proof¶

    We prove the statement by induction on \(n\).

    Base Case: n = 1¶

    By definition of the transition matrix, $$ (Q^1){ij}=Q=P(X_1=j \mid X_0=i). $$ So the statement is true for \(n=1\).

    Inductive Hypothesis¶

    Assume that for some \(n\ge 1\), $$ (Q^n)_{ik}=P(X_n=k \mid X_0=i) $$ for every pair of states \(i,k\).

    Dummy-index warning: here \(i\) and \(j\) are fixed throughout the argument, while \(k\) is a dummy summation index. It changes inside the sum and disappears after summation.

    We must prove that $$ (Q^{n+1}){ij}=P(X=j \mid X_0=i). $$

    Step 1: Expand the Matrix Product¶

    By matrix multiplication, $$ (Q^{n+1}){ij}=(Q^nQ)^M (Q^n)}=\sum_{k=1{ik}Q. $$

    ASCII structure:

    i --n steps--> k --1 step--> j
      sum over all possible k

    Step 2: Apply the Law of Total Probability¶

    To go from \(i\) to \(j\) in \(n+1\) steps, the chain must be in some intermediate state \(k\) after \(n\) steps. Therefore $$ P(X_{n+1}=j \mid X_0=i) = \sum_{k=1}^M P(X_{n+1}=j, X_n=k \mid X_0=i). $$

    Step 3: Apply the Multiplication Rule¶

    For each \(k\), $$ P(X_{n+1}=j, X_n=k \mid X_0=i) = P(X_{n+1}=j \mid X_n=k, X_0=i)\,P(X_n=k \mid X_0=i). $$

    Step 4: Use the Markov Property¶

    By the Markov property, $$ P(X_{n+1}=j \mid X_n=k, X_0=i)=P(X_{n+1}=j \mid X_n=k). $$

    Because the chain is time-homogeneous, this equals $$ Q_{kj}. $$

    Step 5: Use the Inductive Hypothesis¶

    By the inductive hypothesis, $$ P(X_n=k \mid X_0=i)=(Q^n)_{ik}. $$

    Substitute everything: $$ P(X_{n+1}=j \mid X_0=i) = \sum_{k=1}^M (Q^n){ik}Q. $$

    But the right-hand side is exactly $$ (Q^{n+1})_{ij}. $$

    So $$ (Q^{n+1}){ij}=P(X=j \mid X_0=i). $$

    This completes the induction.

    Intuition After the Proof¶

    Every \((n+1)\)-step path from \(i\) to \(j\) must pass through one intermediate state \(k\) after \(n\) steps. The probability of one route is “probability to reach \(k\)” times “probability to leave \(k\) for \(j\),” and then we add over all possible \(k\).

    What Would Break Without the Markov Property?¶

    If the next-step probability still depended on older history, then $$ P(X_{n+1}=j \mid X_n=k, X_0=i) $$ would not simplify to a function of \(k\) alone. The matrix product would no longer encode the correct probability.

    What Would Break Without Time-Homogeneity?¶

    If the transition probabilities changed with time, then the last factor would depend on the step number, and we would no longer have a single matrix power describing the chain.

    Phenomenon Metadata¶

    Element Purpose
    Structural Signature An \((n+1)\)-step move splits at an intermediate state \(k\)
    Core Invariant The intermediate state fully summarizes the past at the split point
    Compression Handle “Reach k, then leave k”
    Boundary / Failure Mode If the next move depends on older history or on time, the product formula fails
    Phenomenon Web Direct precursor to the Chapman-Kolmogorov Equation

    The Chapman-Kolmogorov Equation¶

    For nonnegative integers \(m\) and \(n\), $$ q_{ij}^{(m+n)}=\sum_{k=1}^M q_{ik}^{(m)}q_{kj}^{(n)}. $$

    In matrix form, $$ Q^{m+n}=Q^mQ^n. $$

    Scouting Report¶

    What to look for: a multi-step transition broken at an intermediate time.

    Why you care: this equation is the probabilistic reason matrix multiplication works for Markov chains.

    Intuition Before the Proof¶

    To go from \(i\) to \(j\) in \(m+n\) steps, the chain must be in some state \(k\) after \(m\) steps, and then it must travel from \(k\) to \(j\) in the remaining \(n\) steps. Since \(k\) is unknown, we sum over all possible states.

    ASCII structure:

    time 0            time m                       time m+n
 i  ---------------->  k  ------------------------->  j
       m steps                    n steps

sum over all possible k

    Full Proof¶

    Fix states \(i\) and \(j\).

    Step 1: Partition by the Intermediate State¶

    At time \(m\), the chain must be in exactly one state. So the events \(\{X_m=k\}\) for \(k=1,\ldots,M\) form a partition. Hence $$ P(X_{m+n}=j \mid X_0=i) = \sum_{k=1}^M P(X_{m+n}=j, X_m=k \mid X_0=i). $$

    Dummy-index warning: \(k\) is only a temporary summation variable. \(i\) and \(j\) are fixed.

    Step 2: Apply the Multiplication Rule¶

    For each \(k\), $$ P(X_{m+n}=j, X_m=k \mid X_0=i) = P(X_{m+n}=j \mid X_m=k, X_0=i)\,P(X_m=k \mid X_0=i). $$

    Step 3: Use the Markov Property¶

    The Markov property gives $$ P(X_{m+n}=j \mid X_m=k, X_0=i) = P(X_{m+n}=j \mid X_m=k). $$

    Step 4: Use Time-Homogeneity¶

    Since the chain is time-homogeneous, $$ P(X_{m+n}=j \mid X_m=k)=q_{kj}^{(n)}. $$ Also, $$ P(X_m=k \mid X_0=i)=q_{ik}^{(m)}. $$

    Step 5: Substitute¶

    So $$ P(X_{m+n}=j \mid X_0=i) = \sum_{k=1}^M q_{ik}^{(m)}q_{kj}^{(n)}. $$

    Therefore, $$ q_{ij}^{(m+n)}=\sum_{k=1}^M q_{ik}^{(m)}q_{kj}^{(n)}. $$

    Intuition After the Proof¶

    This is just “break the trip into two legs.” The state at time \(m\) is hidden, so we sum over every possible intermediate state.

    Why We Multiply (The AND Rule)¶

    To travel from \(i\) to \(j\) through a specific intermediate state \(k\), both of these must happen: 1. \(i \to k\) in \(m\) steps 2. \(k \to j\) in \(n\) steps

    So the probability of that route is the product $$ q_{ik}^{(m)}q_{kj}^{(n)}. $$

    Why We Sum (The OR Rule)¶

    The intermediate state \(k\) could be any state in the state space. These possibilities are mutually exclusive, so we add the probabilities over all possible \(k\).

    The Linear Algebra Connection¶

    The quantity $$ \sum_{k=1}^M q_{ik}^{(m)}q_{kj}^{(n)} $$ is exactly the \((i,j)\) entry of the matrix product \(Q^mQ^n\).

    That is why matrix multiplication matches the probabilistic structure of Markov chains so perfectly.

    Worked Example: Computing Q2 and 2-Step Probabilities¶

    Consider $$ Q= \begin{pmatrix} 0.5 & 0.4 & 0.1 \ 0.3 & 0.2 & 0.5 \ 0.2 & 0.3 & 0.5 \end{pmatrix}. $$

    Computing Q2 Explicitly¶

    Entry (1,1)¶

    \[ (Q^2)_{11}=(0.5)(0.5)+(0.4)(0.3)+(0.1)(0.2)=0.25+0.12+0.02=0.39. \]

    Entry (1,2)¶

    \[ (Q^2)_{12}=(0.5)(0.4)+(0.4)(0.2)+(0.1)(0.3)=0.20+0.08+0.03=0.31. \]

    Entry (1,3)¶

    \[ (Q^2)_{13}=(0.5)(0.1)+(0.4)(0.5)+(0.1)(0.5)=0.05+0.20+0.05=0.30. \]

    Entry (2,1)¶

    \[ (Q^2)_{21}=(0.3)(0.5)+(0.2)(0.3)+(0.5)(0.2)=0.15+0.06+0.10=0.31. \]

    Entry (2,2)¶

    \[ (Q^2)_{22}=(0.3)(0.4)+(0.2)(0.2)+(0.5)(0.3)=0.12+0.04+0.15=0.31. \]

    Entry (2,3)¶

    \[ (Q^2)_{23}=(0.3)(0.1)+(0.2)(0.5)+(0.5)(0.5)=0.03+0.10+0.25=0.38. \]

    Entry (3,1)¶

    \[ (Q^2)_{31}=(0.2)(0.5)+(0.3)(0.3)+(0.5)(0.2)=0.10+0.09+0.10=0.29. \]

    Entry (3,2)¶

    \[ (Q^2)_{32}=(0.2)(0.4)+(0.3)(0.2)+(0.5)(0.3)=0.08+0.06+0.15=0.29. \]

    Entry (3,3)¶

    \[ (Q^2)_{33}=(0.2)(0.1)+(0.3)(0.5)+(0.5)(0.5)=0.02+0.15+0.25=0.42. \]

    So $$ Q^2= \begin{pmatrix} 0.39 & 0.31 & 0.30 \ 0.31 & 0.31 & 0.38 \ 0.29 & 0.29 & 0.42 \end{pmatrix}. $$

    Each row sums to \(1\), as expected.

    Scenario A: Starting from State 1, Reach State 3 in 2 Steps¶

    By the theorem, $$ P(X_2=3 \mid X_0=1)=(Q^2)_{13}=0.30. $$

    By path decomposition, $$ q_{13}^{(2)}=q_{11}q_{13}+q_{12}q_{23}+q_{13}q_{33}. $$

    Substitute: $$ q_{13}^{(2)}=(0.5)(0.1)+(0.4)(0.5)+(0.1)(0.5)=0.05+0.20+0.05=0.30. $$

    The three paths are: - \(1 \to 1 \to 3\), probability \(0.05\) - \(1 \to 2 \to 3\), probability \(0.20\) - \(1 \to 3 \to 3\), probability \(0.05\)

    The most likely route is through state \(2\).

    Scenario B: Starting from State 2, Reach State 3 in 2 Steps¶

    Again, $$ P(X_2=3 \mid X_0=2)=(Q^2)_{23}=0.38. $$

    By path decomposition, $$ q_{23}^{(2)}=q_{21}q_{13}+q_{22}q_{23}+q_{23}q_{33}. $$

    Substitute: $$ q_{23}^{(2)}=(0.3)(0.1)+(0.2)(0.5)+(0.5)(0.5)=0.03+0.10+0.25=0.38. $$

    The three paths are: - \(2 \to 1 \to 3\), probability \(0.03\) - \(2 \to 2 \to 3\), probability \(0.10\) - \(2 \to 3 \to 3\), probability \(0.25\)

    The most likely route is through state \(3\).

    Intuition After the Example¶

    The example makes the matrix interpretation concrete: each entry of \(Q^2\) is a sum over all possible hidden middle states. The two-step probability is not the square of a single entry. It is a sum of route probabilities.

    Common Mistakes¶

    Mistake 1: Markov Does Not Mean Independent¶

    A Markov chain is not generally independent across time. The future often depends strongly on the present.

    For example, $$ Q= \begin{pmatrix} 0 & 1 \ 1 & 0 \end{pmatrix} $$ is perfectly Markov but maximally dependent: the chain alternates deterministically.

    Mistake 2: Rows and Columns Are Not Interchangeable¶

    Rows represent the current state. Columns represent the next state.

    Transposing the matrix changes the meaning completely. It gives backward-looking probabilities rather than forward transition probabilities.

    Mistake 3: Qn Does Not Mean nQ¶

    \(Q^n\) means repeated matrix multiplication, not scalar multiplication of every entry by \(n\).

    For example, if $$ Q= \begin{pmatrix} 0.5 & 0.5 \ 0.5 & 0.5 \end{pmatrix}, $$ then \(Q^2=Q\), but $$ 2Q= \begin{pmatrix} 1 & 1 \ 1 & 1 \end{pmatrix}, $$ which is not stochastic.

    Mistake 4: Future Independent of Past Does Not Mean Future Independent of Present¶

    The Markov property says the future is independent of the past given the present. It does not say the future is independent of the present.

    That distinction matters whenever you compute a conditional probability versus a marginal one.

    Notation Trap¶

    When you see $$ q_{ij}^{(n)}, $$ the superscript \((n)\) indicates an entry of \(Q^n\). It is not the scalar power \((q_{ij})^n\).

    Dummy-index warning: in sums like $$ \sum_{k=1}^M q_{ik}^{(m)}q_{kj}^{(n)}, $$ the index \(k\) is temporary. It is not a fixed state like \(i\) or \(j\).

    Looking Ahead¶

    If an initial distribution is \(t\), then the distribution after \(n\) steps is $$ tQ^n. $$

    A natural next question is whether the chain stabilizes as \(n\) grows large. For finite irreducible aperiodic chains, the answer is yes: the distribution converges to a stationary distribution \(s\) satisfying $$ sQ=s. $$

    This is a left eigenvector equation with eigenvalue \(1\).

    Not every chain converges. Periodic chains can oscillate forever, and reducible chains can behave differently depending on where they start.

    Later topics include hitting times, absorption, expected return times, and reversibility.

    Phenomenon Metadata¶

    Element Purpose
    Structural Signature Repeated application of the same transition matrix to an initial distribution
    Core Invariant Long-run behavior is encoded by fixed points of the update rule
    Compression Handle “Stationary means sQ = s”
    Boundary / Failure Mode Periodicity or reducibility can prevent convergence
    Phenomenon Web See the Transition Matrix Q block and the n-Step Transition Matrix Qn block
